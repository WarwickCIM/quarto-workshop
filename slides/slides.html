Quarto

Preamble

  • Literate programming is important
  • Lots of different solutions, some are language specific

R: Rmarkdown

  • Markdown with configurable code blocks
  • Focus on R (but does support other languages see here)
  • Can compile to other formats:
    • natively
    • bookdown, blogdown, …

Python: Jupyter notebook

  • Single JSON file containing code, text and output
  • Kernels for many programming languages
  • Can compile to other formats:
    • Jupyter book, nbconvert, …

About quarto

  • “Open-source scientific and technical publishing system built on Pandoc” - quarto.org
  • Developed by Posit (RStudio) as Rmarkdown’s next generation. “Code output via integration with Jupyter, Knitr, and Observable” - GitHub Repo
  • Batteries included (“articles, reports, presentations, websites, blogs, and books in HTML, PDF, MS Word, ePub, and more”) - quarto.org
  • Code chunks in .qmd markdown files are evaluated by the language engine, and the document is built using Pandoc with reasonable default formatting

Rmarkdown’s next generation

  • One qmd file -> parser -> one or more outputs
  • 2 files:
    • qmd containing the “source” (configuration, text and code)
    • * containing the output (pdf, html, docx…)
  • Syntax:
    • Config: yaml, metadata
    • Rich text: markdown on steroids (variable text -inline code-, callouts, mermaid diagrams…)
    • Code: code blocks (they can have special comments for configs, )
  • One qmd file, one or more outputs

Features: Programming languages

  • Works with R, Python, Julia, Observable
    • Can combine different languages in a single document

Features: Integrations

  • getting started
    • RStudio
    • Visual Studio Code/VS Codium
    • Jupyter

Features: Outputs

  • Diverse outputs
    • Formats: pdf, html, epub, docx…
    • Media: articles, presentations (like this one!), blogs, websites, books…

Features: Authoring

  • Authoring tools: scientific markdown, including equations, citations, crossrefs, bibliographical references, figure panels, callouts, advanced layout, and more.
  • File format qmd is a text file that doesn’t include output. Human readable and good for version control.

Use cases

Overview

  1. Pick a case from the list you would like to develop during this workshop
  2. Create groups of people who want to work on the same case and assign roles:
  3. Work on your case and document your progress!

This assumes you have already:

  1. Installed quarto in your machine (instructions here)
  2. read the case studies and you have a preference for at least one of them)

Use cases and groups creation

  1. There’s a designated areas for every use case
  2. Move to the designated area, according to the use case you’d like to work on
  3. Meet and greet
  4. Assign roles:
    1. Documentator(s) (at least one person)
    2. Spokesperson/representative (one person, only)

This assumes you have already read the case studies and you have a preference for at least one of them.

If you haven’t, please join one of the existing groups (or create a new one) whenever you’re ready.

Work on your case

  1. Install quarto’s binaries: https://quarto.org/docs/get-started/
  2. Fork this repository
  3. Create a new folder for your case under cases/
  4. Copy the template from cases/template/documentation_template.qmd to your folder, which will be used to document the process
  5. Make any changes needed
  6. Commit any changes
  7. Submit Pull Request to the Workshop’s GitHub repo

Important

Do not forget about documenting the process!
